# 梦幻西游端游师门任务自动化系统 - 需求文档

## 一、项目概述

### 1.1 项目背景
师门任务是《梦幻西游》端游中最基础的日常玩法，玩家加入门派后每天可完成 20 个师门任务获得双倍奖励，前 20 个任务完成后还可继续完成至 100 环。本系统旨在实现师门任务的自动化完成。

### 1.2 项目目标
- 实现师门任务全流程自动化
- 支持所有等级段（20-155 级）的师门任务
- 支持所有任务类型的自动处理
- 提供稳定、高效的自动化解决方案

## 二、师门任务机制分析

### 2.1 任务刷新机制
- **刷新时间**：每天 24:00 刷新
- **双倍奖励**：每天前 20 个任务可获得双倍奖励
- **三倍奖励**：服务器开放一年后，20-39 级玩家前 20 个任务为 3 倍奖励
- **轮次**：10 环任务为一轮
- **取消机制**：领取任务 5 分钟后可找师傅取消任务

### 2.2 等级限制
- **20-39 级**：可正常领取
- **≥40 级**：当前经验 < 升级经验×10 且当前经验 < 1 亿时可领取
- **≥100 级**：第 61-120 环金钱奖励为正常的 50%，第 121 环开始金钱奖励为 0

### 2.3 奖励机制
- 任务奖励不受双倍/精修时间（3 倍）影响
- 战斗死亡计算正常死亡，有死亡损失
- 每日 20 次、每周 100 次、每月 400 次均有储备金奖励

### 2.4 100 级以上师门流程（实现视角）
> 以下描述以常见端游版本逻辑为归纳，**具体文案、NPC 名、是否含战斗以当前游戏版本为准**，上线前需对照游戏内任务说明做一次核对。

- **共性主循环**：回门派 → 与师傅对话领任务 → 根据任务栏/对话确定类型与目标 → 执行子流程 → 回师傅处交任务 → 记录环数与奖励。
- **寻物**：任务栏会写明物品名（及数量）。高等级常见为环装、指定三药/烹饪等。自动化路径一般为：先搜背包/任务包裹 → 若无则按策略购买（商会、摊位、固定商店 NPC）→ 回师傅交付。
- **捕捉**：任务栏写明召唤兽种类。优先商会检索购买；若配置允许再考虑野外遇怪捕捉（耗时长、不确定性高，建议默认关闭或仅作兜底）。
- **示威**：需前往**指定地图/坐标**（或指定区域），在目标点附近使用**本门派特定技能**（或按游戏当前机制操作）。需配置：示威地图关键词 → 坐标或区域、技能快捷键/点击方案、完成判定（界面变化/OCR/等待时长）。
- **支援 / 找人**（100+ 常见）：前往某地图寻找**指定门派弟子 NPC**（或特定 NPC）并对话。本质是「寻路 + 对话树」。难点在于 NPC 名称与地图的映射表维护，以及对话选项的固定路径。
- **与低等级差异小结**：100+ 减少了巡逻、送信等类型（以当前版本为准），**寻物单价高、示威/找人跨地图多**，对「寻路稳定性、资金上限、失败重试」要求更高。

### 2.5 地图与路线策略（不要求一次性背下「全地图」）
- **原则**：程序不依赖人工背诵所有地图连通关系，而是依赖 **游戏内建寻路 + 可配置传送/坐标库**。
- **推荐能力组合**（按优先级）：
  1. **任务追踪**：点击任务栏自动寻路（若客户端支持且坐标稳定，可减少 OCR 依赖）。
  2. **ALT+G / 小地图坐标寻路**：对已知的 `(地图名, x, y)` 直接输入。
  3. **导标旗、飞行符、合成旗、驿站**：在配置中维护「常用点 → 使用的道具与操作步骤」（脚本化点击或快捷键）。
  4. **NPC 传送**：长安罗道人、各地图传送员等，维护「从 A 到 B 的 NPC 链」或最短操作序列（随版本变化大，需可热更新）。
- **「每个地图的路线」在文档中的落点**：不单独成章背诵，而是沉淀为 **NPC 字典、传送规则、示威/找人目的地表**；缺数据时由你方在游戏里实测后填入配置文件。

## 三、任务类型分类

### 3.1 20-39 级任务类型
| 任务类型 | 说明 | 处理方式 |
|---------|------|---------|
| 送信任务 | 给指定 NPC 送信 | 自动寻路至 NPC，对话交付 |
| 寻物任务 | 寻找指定物品（0 级武器装备、一级药、杂货、包子） | 从 NPC 购买/背包使用 |
| 门派巡逻 | 在门派指定地点巡逻 | 自动寻路至巡逻点，点击巡逻 |

### 3.2 40-69 级任务类型
| 任务类型 | 说明 | 处理方式 |
|---------|------|---------|
| 寻物任务 | 寻找指定物品（武器装备、药品、杂货等） | 从 NPC 购买/背包使用/商会购买 |
| 门派巡逻 | 在门派指定地点巡逻 | 自动寻路至巡逻点，点击巡逻 |
| 捕捉任务 | 捕捉指定召唤兽 | 长安商会购买/野外捕捉 |

### 3.3 70-99 级任务类型
| 任务类型 | 说明 | 处理方式 |
|---------|------|---------|
| 寻物任务 | 寻找指定物品（含烹饪、三级药） | 从 NPC 购买/背包使用/玩家摊位 |
| 捕捉任务 | 捕捉指定召唤兽 | 长安商会购买/野外捕捉 |
| 示威任务 | 在指定地点展示门派技能 | 自动寻路至地点，使用技能 |

### 3.4 100 级以上任务类型
| 任务类型 | 说明 | 处理方式 |
|---------|------|---------|
| 寻物任务 | 寻找高级物品（环装、高级烹饪、高级药品） | NPC 购买/玩家摊位/自建商店 |
| 捕捉任务 | 捕捉高级召唤兽 | 长安商会购买/玩家购买 |
| 示威任务 | 在指定地点展示门派技能 | 自动寻路至地点，使用技能 |
| 找人任务 | 寻找指定 NPC 或玩家 | 自动寻路至目标，对话 |

### 3.5 任务类型 → 自动化执行器对照（需求追溯）
| 任务类型 | 识别关键词（示例，需按真实任务栏文案校准） | 执行器 | 关键依赖数据 |
|---------|------------------------------------------|--------|-------------|
| 寻物 | 物品名、数量、「找来」「上交」等 | 寻物模块 | 物品获取策略、价格上限、商店/摊位规则 |
| 巡逻 | 「巡逻」、门派场景内 | 巡逻模块 | 各门派巡逻点坐标 |
| 捕捉 | 召唤兽名、「捉」「抓」 | 捕捉模块 | 商会搜索流程、可选野外遇怪配置 |
| 示威 | 地图名、坐标、「示威」「展示」 | 示威模块 | 示威点坐标或区域、技能释放方式 |
| 送信 | NPC、「信」、低等级 | 送信/找人模块 | NPC 字典 |
| 找人/支援 | 门派弟子、NPC 名、地图 | 送信/找人模块 | NPC 字典、对话选项顺序 |

## 四、系统功能需求

### 4.1 核心功能模块

#### 4.1.1 任务领取模块
- **功能描述**：自动找门派师傅领取师门任务
- **输入**：门派名称、师傅位置坐标
- **处理流程**：
  1. 自动寻路至门派师傅处
  2. 与师傅对话
  3. 选择领取师门任务
  4. 识别任务类型和任务要求
- **输出**：任务信息（任务类型、目标物品/NPC/怪物、数量等）

#### 4.1.2 任务识别模块
- **功能描述**：识别当前任务类型和具体要求，并输出可供调度器使用的结构化任务数据。
- **输入**（可多源融合，按实现阶段取舍）：
  - 任务栏 / 任务追踪区域截图（主路径）
  - 与师傅对话窗口截图（领任务后首屏）
  - 可选：任务追踪点击后的自动寻路目的地提示
- **处理流程（建议）**：
  1. **ROI 裁剪**：固定窗口客户区比例或模板定位任务栏区域，减少 OCR 噪声。
  2. **OCR**：PaddleOCR / 百度 OCR 等；对游戏字体可准备**专用字典**（物品名、地图名、NPC 名、门派名）。
  3. **规则分类**：按关键词与正则匹配任务类型（见 3.5）；无法匹配时标记 `unknown` 进入人工/日志分支。
  4. **实体抽取**：
     - 寻物：物品名、数量、等级段（若有）
     - 捕捉：召唤兽名
     - 示威/找人：地图名、坐标（`xxx, yyy` 或「X Y」多种格式需兼容）、NPC 名
  5. **归一化**：将 OCR 结果通过**别名表**映射到配置中的标准名（如繁简、错字、简称）。
  6. **置信度**：低置信度时重试截图、换引擎或中止并告警。
- **输出**：结构化任务数据（见第五节 `TaskInfo` 扩展字段）
- **非目标（第一版可不做）**：读内存、封包解析等侵入客户端的方式；本文档仅描述**屏幕自动化**路径。

#### 4.1.3 寻物任务处理模块
- **功能描述**：自动完成寻物类任务
- **子功能**：
  - **NPC 购买**：自动寻路至 NPC，打开商店，购买物品
  - **背包使用**：检查背包是否有任务物品，直接使用
  - **商会购买**：打开长安商会，搜索并购买召唤兽/物品
  - **摊位购买**：自动查找玩家摊位，购买所需物品
- **所需数据**：
  - NPC 位置字典
  - 物品分类字典（武器装备、药品、烹饪、杂货等）
  - 价格上限配置

#### 4.1.4 巡逻任务处理模块
- **功能描述**：自动完成门派巡逻任务
- **处理流程**：
  1. 读取门派巡逻点坐标
  2. 自动寻路至巡逻点
  3. 执行巡逻动作（点击巡逻按钮）
  4. 等待巡逻完成
- **所需数据**：各门派巡逻点坐标

#### 4.1.5 捕捉任务处理模块
- **功能描述**：自动完成召唤兽捕捉任务
- **处理流程**：
  1. 识别需要捕捉的召唤兽类型
  2. 优先从长安商会购买
  3. 商会无货时野外捕捉（可选）
  4. 将召唤兽交予师傅
- **所需数据**：召唤兽名称字典、商会位置

#### 4.1.6 示威任务处理模块
- **功能描述**：自动完成门派示威任务
- **处理流程**：
  1. 识别示威地点
  2. 自动寻路至地点
  3. 使用门派技能
  4. 等待任务完成
- **所需数据**：示威地点坐标、门派技能配置

#### 4.1.7 送信/找人任务处理模块
- **功能描述**：自动完成送信和找人任务
- **处理流程**：
  1. 识别目标 NPC 或人物
  2. 自动寻路至目标
  3. 对话交付任务
- **所需数据**：NPC 位置字典

#### 4.1.8 任务交付模块
- **功能描述**：将完成任务交予门派师傅
- **处理流程**：
  1. 自动寻路至师傅处
  2. 对话交付任务
  3. 领取奖励
  4. 记录任务完成情况

### 4.2 辅助功能模块

#### 4.2.1 自动寻路模块
- **功能描述**：实现游戏内自动寻路
- **实现方式**：
  - 基于游戏内寻路系统（ALT+G 输入坐标）
  - 基于地图导航 NPC
  - 基于预设路线

#### 4.2.2 背包管理模块
- **功能描述**：管理背包物品，整理任务物品
- **功能**：
  - 检查背包物品
  - 整理背包空间
  - 识别任务物品

#### 4.2.3 资金管理模块
- **功能描述**：管理任务资金，确保购买能力
- **功能**：
  - 检查当前金钱
  - 设置购买价格上限
  - 资金不足时提醒

#### 4.2.4 异常处理模块
- **功能描述**：处理任务过程中的异常情况
- **异常类型**：
  - 找不到 NPC
  - 物品购买失败
  - 任务超时
  - 战斗死亡
  - 网络断开
- **处理策略**：
  - 重试机制
  - 取消任务（5 分钟后）
  - 记录日志
  - 用户通知

#### 4.2.5 日志记录模块
- **功能描述**：记录任务完成情况
- **记录内容**：
  - 任务编号（第几环）
  - 任务类型
  - 完成时间
  - 奖励内容
  - 异常情况

## 五、数据结构设计

### 5.1 任务信息结构
```python
class TaskInfo:
    task_number: int          # 第几环
    task_type: str            # 任务类型（寻物/巡逻/捕捉/示威/送信/找人）
    target_name: str          # 目标名称（物品/NPC/怪物）
    target_quantity: int      # 目标数量
    target_level: int         # 物品等级（如适用）
    location: str             # 地图名（标准化后）
    coordinates: tuple        # 坐标 (x, y)，可选
    raw_text: str             # OCR 原始文本，便于调试
    confidence: float         # 识别置信度
    deadline: datetime        # 任务截止时间（若可解析）
```

## 七、技术实现要点

### 7.1 图像识别
- **OCR 文字识别**：使用百度 OCR 或 PaddleOCR 识别任务文字
- **模板匹配**：识别游戏界面按钮、NPC 头像等
- **颜色识别**：识别任务追踪框、小地图标记

### 7.2 自动化控制
- **鼠标控制**：pyautogui/pynput 实现鼠标点击、拖动
- **键盘控制**：pyautogui/pynput 实现键盘按键
- **窗口管理**：pywin32 实现窗口定位、激活

### 7.3 寻路实现
- **坐标寻路**：ALT+G 输入坐标寻路
- **NPC 寻路**：通过寻路 NPC 自动寻路
- **地图切换**：处理地图传送点

### 7.4 配置管理
- **配置文件**：JSON/YAML格式存储配置
- **热更新**：支持运行时更新配置
- **多门派支持**：不同门派独立配置

## 九、风险评估

### 9.1 技术风险
- **OCR 识别准确率**：游戏字体识别可能不准确
  - *应对*：多 OCR 引擎备选，模板匹配辅助
- **游戏更新**：游戏界面变化导致识别失败
  - *应对*：配置化设计，便于快速适配
- **反作弊机制**：可能被检测为外挂
  - *应对*：模拟人工操作，添加随机延迟

### 9.2 使用风险
- **封号风险**：使用自动化可能被处罚
  - *应对*：提供使用说明，建议适度使用
- **依赖风险**：过度依赖导致手动操作能力下降
  - *应对*：仅作为辅助工具定位

## 十一、可实现程度、协作边界与需你方提供的资料

### 11.1 在「屏幕自动化 + 配置驱动」前提下，可交付的能力层级

| 层级 | 内容 | 说明 |
|------|------|------|
| L1 | 工程骨架 | 窗口绑定、截图 ROI、键鼠封装、日志、配置加载 |
| L2 | 单门派闭环 | 固定分辨率下：领任务 → OCR 识别 → 一种任务类型（如仅寻物或仅巡逻）→ 交任务 |
| L3 | 多类型路由 | 任务类型分类器 + 各执行器雏形，覆盖你方已提供坐标/对话路径的任务 |
| L4 | 多门派与传送 | 导标旗/飞行符等流程脚本化，NPC 字典与别名表较完整 |
| L5 | 高鲁棒 | 异常重试、取消任务、资金与背包策略、UI 微改版可快速调配置恢复 |

**说明**：我能协助完成 **代码、配置结构、识别与流程设计、与现有 jingshi 等目录的对接思路**；**无法**保证绕过游戏安全策略、也无法代你承担封号等风险。具体能到 L几，取决于你提供的**真实截图、分辨率、门派与任务样本文本**。

### 11.2 建议你方一次性准备好的资料（优先级从高到低）

1. **环境与约束**：游戏分辨率（窗口/全屏）、是否多开、操作系统、是否允许使用外部 OCR API（联网）。
2. **账号与角色**：等级区间（如 109 / 129 / 175）、门派、常用上线地图；若只做 100+ 可明确范围以缩小字典。
3. **截图包**（最关键）：
   - 任务栏清晰截图若干张（每种任务类型至少 2～3 张，含不同物品/NPC/地图文案）
   - 领任务、交任务时师傅对话截图
   - 商会、摊位购买界面截图（若要做自动买）
4. **实测数据**：你正在玩的版本里，**师傅名、门派地图名、师傅坐标**；示威常见地图与坐标样本；常用传送（旗子点、飞行符落点）的操作习惯。
5. **策略参数**：单次寻物金钱上限、是否允许摊位扫货、是否允许野外抓宠、每环超时时间、失败是否自动取消任务等。
6. **可选**：与 `jingshi` 等现有模块的调用约定（同进程 import 还是子进程、配置路径）。

### 11.3 文档与实现的维护约定
- 游戏小版本更新后，优先核对：**任务栏文案、按钮位置、师傅坐标**；通过改配置而非改代码修复为佳。
- 新增任务类型时：先补 **3.5 路由表** 与 **识别规则**，再实现执行器。
