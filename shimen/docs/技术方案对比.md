# 师门任务识别技术方案对比

## 📊 总览

| 识别对象 | 推荐方案 | 理由 | 开发周期 |
|---------|---------|------|---------|
| **导标旗** | 模板匹配 + OCR | 界面规则，位置固定 | 1-2 天 |
| **NPC** | YOLO + OCR | 会移动，位置不固定 | 5-7 天 |
| **任务文字** | OCR | 纯文字识别 | 1 天 |
| **小地图标记** | 颜色识别 + 模板 | 固定样式 | 1 天 |
| **战斗场景** | YOLO | 高速移动目标 | 3-5 天 |

---

## 🎯 导标旗识别方案

### 技术选择：**模板匹配 + OCR** ✅

#### 为什么不用 YOLO？

| 因素 | 分析 | 结论 |
|------|------|------|
| 位置 | ALT+T 界面固定位置 | ✅ 不需要目标检测 |
| 变化 | 导标旗图标固定 | ✅ 模板匹配足够 |
| 关键信息 | 导标旗名称（文字） | ✅ OCR 是核心 |
| 开发成本 | YOLO 需训练数据集 | ❌ 成本高 |
| 识别速度 | 模板匹配极快 | ✅ 性能优秀 |

#### 实现架构

```
┌─────────────────────────────────────┐
│  按 ALT+T 打开导标旗界面             │
├─────────────────────────────────────┤
│  模板匹配定位插槽位置                │
│  - 固定位置或模板检测               │
├─────────────────────────────────────┤
│  OCR 识别导标旗名称                 │
│  - 百度 OCR / PaddleOCR             │
├─────────────────────────────────────┤
│  配置文件存储坐标信息                │
│  - flags_config.json                │
└─────────────────────────────────────┘
```

#### 代码示例

```python
# 导标旗识别（已实现）
from navigation.flag_recognition import FlagRecognitionSystem

flag_system = FlagRecognitionSystem(ocr_service)

# 打开界面
flag_system.open_flag_interface(window_manager)

# 识别
screenshot = window_manager.capture_screen()
flags = flag_system.detect_and_recognize(screenshot)

# 输出：[{'name': '长安城驿站', 'position': {...}}, ...]
```

## 🎯 NPC 识别方案

### 技术选择：**YOLO 目标检测 + OCR** ✅

#### 为什么必须用 YOLO？

| 因素 | 分析 | 结论 |
|------|------|------|
| 位置 | NPC 会移动，位置不固定 | ✅ 必须目标检测 |
| 角度 | NPC 朝向不同 | ✅ YOLO 抗角度变化 |
| 遮挡 | 被其他角色遮挡 | ✅ YOLO 鲁棒性强 |
| 种类 | 34 种不同 NPC | ✅ 多类别检测 |
| 光照 | 白天/夜晚差异 | ✅ 抗光照变化 |

#### 实现架构

```
┌─────────────────────────────────────┐
│  游戏画面截图                        │
├─────────────────────────────────────┤
│  YOLO 检测 NPC 位置                  │
│  - 输出：[(x,y,w,h,class,conf),...] │
├─────────────────────────────────────┤
│  OCR 识别头顶名字（辅助）            │
│  - 确认 NPC 身份                     │
├─────────────────────────────────────┤
│  匹配目标 NPC                        │
│  - 根据任务需求选择                 │
├─────────────────────────────────────┤
│  导航至 NPC 位置                     │
│  - 点击或 ALT+G 寻路                │
└─────────────────────────────────────┘
```

#### 代码示例

```python
# NPC 识别（已实现）
from navigation.npc_recognition import NPCRecognitionSystem

npc_system = NPCRecognitionSystem(ocr_service)

# 查找门派师傅
screenshot = window_manager.capture_screen()
master = npc_system.find_npc(screenshot, "门派师傅")

if master:
    # 导航到 NPC
    npc_system.navigate_to_npc(window_manager, "门派师傅")
```

#### 训练流程

```
1. 数据收集（2-3 天）
   └─→ 游戏截图 2000-5000 张

2. 数据标注（2-3 天）
   └─→ LabelImg 标注 NPC 位置

3. 模型训练（1-2 天）
   └─→ YOLOv8 训练 100 epochs

4. 模型优化（持续）
   └─→ 困难样本挖掘、调参
```

## 🎯 任务文字识别方案

### 技术选择：**OCR** ✅

#### 方案对比

| 方案 | 准确率 | 速度 | 开发成本 |
|------|--------|------|---------|
| 百度 OCR | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

| Tesseract | ⭐⭐⭐ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ |

#### 推荐：**PaddleOCR**

```python
from paddleocr import PaddleOCR

ocr = PaddleOCR(use_angle_cls=True, lang='ch')
```

## 🎮 实战建议

### 导标旗识别
1. **优先实现**：模板匹配方案
2. **配置化**：导标旗坐标可配置
3. **容错**：识别失败时手动补充

### NPC 识别
1. **数据优先**：先收集 500 张截图
2. **重点 NPC**：优先标注师傅、师兄等核心 NPC
3. **迭代优化**：根据识别效果持续改进

### 混合策略
1. **导标旗优先**：优先使用导标旗传送
2. **坐标寻路备用**：导标旗无法到达时使用
3. **NPC 检测兜底**：最后使用 YOLO 检测
