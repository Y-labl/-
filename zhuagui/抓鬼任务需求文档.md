# 梦幻西游端游抓鬼任务自动化系统 - 需求文档

## 一、项目概述

### 1.1 项目背景
抓鬼任务是《梦幻西游》端游中最经典的日常组队玩法之一，玩家组成 5 人队伍，通过战斗击败各种鬼魂获得经验、金钱和物品奖励。相比师门任务，抓鬼任务涉及的 NPC 更少（仅 4 个），流程更简单，是理想的自动化入门项目。

### 1.2 项目目标
- 实现抓鬼任务全流程自动化
- 路线相关 4 个 NPC + 战斗中 2 类目标（与 YOLO 类别配置一致）
- 支持自动组队、自动战斗、自动领奖
- 开发周期短，快速验证技术可行性

## 二、抓鬼任务机制分析

### 2.1 任务流程
```
1. 组建队伍（5 人）
   ↓
2. 找马副将领取双倍奖励（长安城）
   ↓
3. 飞到驿站
   ↓
4. 找驿站老板传送到大唐国境
   ↓
5. 从大唐国境跑到地府
   ↓
6. 找黑无常领取鬼王任务
   ↓
7. 出来找钟馗领取抓鬼任务
   ↓
8. 使用天眼通符（显示真实坐标）
   ↓
9. 识别任务栏（地图 + 坐标）
   ↓
10. 寻路到目标位置
   ↓
11. 击杀鬼/鬼王
   ↓
12. 返回驿站 → 大唐国境 → 地府（循环）
```

### 2.2 任务限制
- **等级要求**：≥ 30 级
- **组队要求**：必须 5 人队伍
- **任务轮数**：最多 10 轮/天
- **时间限制**：每轮任务 10 分钟
- **双倍奖励**：前 20 个鬼可获得双倍奖励

### 2.3 奖励机制
- **经验奖励**：根据等级和鬼的类型
- **金钱奖励**：基础金钱 + 物品奖励
- **物品奖励**：环装、乐器、花、暗器等
- **抓鬼积分**：可在积分使者处兑换物品

### 2.4 流程变体与版本说明（实现前须核对）
> 下列规则随资料片、服务器类型（计时服/免费服等）可能调整，**以你当前客户端与官网说明为准**；自动化侧一律做成**可配置开关**，避免写死在代码里。

- **只做「钟馗抓鬼」还是「黑无常鬼王 + 钟馗」**：文档默认流程含鬼王；若你方只跑普通抓鬼，可在配置中关闭鬼王分支，跳过黑无常相关状态。
- **天眼通符**：用于将假坐标变为真实坐标（若任务机制仍如此）。若角色习惯靠队友天眼、或玩法改版，本模块应可整体关闭，改为仅 OCR 任务栏已有坐标。
- **双倍 / 精修**：马副将领取步骤是否每次执行、是否与队长不同步，由策略配置决定。
- **组队角色**：全队长动（5 开）与「仅队长脚本 + 真人队友」对组队、战斗结束判定、异常处理要求不同，需在需求中选定主场景（见 §十、L5 与 L4 描述）。

### 2.5 地图与路线策略（不要求背诵全地图）
- **主路径**：长安（马副将、驿站）→ 驿站传送大唐国境 → 跑图或飞行至地府 → 黑无常 / 钟馗 → 领任务后 **ALT+G / 任务追踪** 去目标地图坐标 → 战斗 → 回地府交任务（或按当前版本交任务规则）。
- **远距离地图**：依赖 **飞行符、导标旗、合成旗** 等，在配置中维护「从当前地图回地府 / 去常用出口」的操作序列，与师门文档同样采用**配置驱动**，不在正文罗列每张地图走法。

### 2.6 抓鬼前置子流程：长安导标旗 → 马副将 / 驿站（自动化逐步说明）

> 对应实现：`zhuagui/bot/guide_flag.py`、`--ghost-step1`、`--fly-changan`；配置见 `config/bot_default.yaml` 中 `guide_flag`。

以下为**建议状态机**，每一步均可记录日志、截图，便于排查。

| 步骤 | 动作 | 成功判据 | 失败时 |
|------|------|----------|--------|
| S0 | 查找游戏窗口、激活（必要时用户点一下窗口） | 窗口句柄有效 | 重试查找或提示用户 |
| S1 | 按配置按若干次 `ESC`，关闭可能挡在顶层的界面 | — | 可增按次数 |
| S2 | **将系统鼠标移到窗口内指定比例**（默认 `[0.5, 0.5]` 中心），再点任务追踪区一次拿焦点 | 焦点在游戏内 | 用户手动点游戏；终端前台时 `Alt+E` 会发到终端 |
| S3 | 发送 `Alt+E` 打开道具栏（道具行囊） | 见 S4 | 可配置再点背包图标 |
| S4 | 等待 `after_open_inventory_s` | — | — |
| S5 | 在背包 ROI 内用 **导标旗图标模板** 匹配长安红旗 | 得分 ≥ `icon_match_threshold` → 转 S7 | 未找到 → 转 S6 |
| S6 | 按 `ESC`×`esc_before_reopen_inventory` 后回到 S3，循环计数直至 `max_inventory_open_attempts` | 某次 S5 找到 → S7 | 仍无 → S7 用比例坐标 `first_slot_center_norm` 兜底右键 |
| S7 | **缓慢移动**系统鼠标到匹配中心（`move_to_flag_duration_s`），再右键；可选 `align_after_move_to_flag` 再对齐游戏内光标后右键 | 弹出导标地图 | 见 §2.7、§13.3 |
| S8 | 全窗截图，模板匹配地图上的马副将 / 驿站红点 | 得分 ≥ `match_threshold` | 用本机地图界面重截模板或略降阈值 |
| S9 | 左键目标点，等待传送，再 `ESC` 关层 | 地图关闭、角色到位 | 超时重试 |

**玩法要点（与攻略一致）**：合成旗 / 导标旗放在物品栏，**右键**使用；长安旗需在奇门遁甲 3 级等前提下定过长安坐标；马副将处可买旗、领双倍等（与抓鬼链其他步骤配合）。

### 2.7 梦幻西游端游「鼠标漂移 / 双光标」与脚本的对应关系（简要）

端游客户端会绘制**游戏内指针**，而常见自动化库只移动**系统指针**，二者在屏幕上可能不一致，表现为「脚本以为点到了，游戏里指针还在别处」。

**环境侧（玩家可先排查）**（综合官网社区、17173、经验文章等）：

- Windows 鼠标设置中可尝试**关闭「提高指针精确度」**，避免加速曲线干扰重复定位。
- **显卡 / 鼠标驱动**更新；游戏分辨率与桌面缩放（100% / 125%）尽量一致；可尝试窗口 / 全屏 / 无边窗口。
- 部分文章提到客户端分辨率模式（如经典与高清）会改变 UI 缩放，**模板与比例坐标需按当前模式重标定**。

**脚本侧（本仓库策略）**：

1. **入口 `run_bot.py` 调用 `SetProcessDPIAware()`**，减轻 pygetwindow 与 pyautogui 坐标系不一致（高 DPI 屏）。
2. **打开背包前**先移到窗口中心比例（`park_before_inventory_norm`），再点任务区、再 `Alt+E`。
3. **找导标旗**：优先图标模板匹配得到屏幕坐标，**缓慢移动**后右键（`move_to_flag_duration_s`）。
4. **可选** `align_after_move_to_flag`：在缓慢移动后，用 `cursor_align` 在目标附近匹配游戏内指针模板并分象限微调，再右键（背包内指针外观可能变化，需自备 `template_path_alt` 或调低 `cursor.match_threshold`）。
5. 与按键精灵社区常见写法一致的核心思想：**用截图 + 模板得到「游戏内指针」位置，再用相对移动修正**，而不是只信一次 `MoveTo`。

## 四、系统功能需求

### 4.1 核心功能模块

#### 4.1.1 组队模块
- **功能描述**：自动组建或加入抓鬼队伍
- **子功能**：
  - 创建队伍（当队长）
  - 申请加入队伍（当队员）
  - 邀请玩家入队
  - 等待队伍满员
- **识别需求**：
  - 队伍界面识别
  - 玩家等级识别
  - 队长标识识别

#### 4.1.2 双倍领取模块
- **功能描述**：自动使用飞行棋找马副将领取双倍奖励
- **处理流程**：
  1. 自动使用飞行棋寻路至长安城马副将处
  2. 与马副将对话
  3. 选择领取双倍奖励
  4. 确认领取
- **识别需求**：
  - 马副将 NPC 识别（YOLO）
  - 对话界面识别（模板匹配）
  - 双倍奖励界面识别

#### 4.1.3 驿站传送模块
- **功能描述**：自动使用飞行棋找驿站老板传送到大唐国境
- **处理流程**：
  1. 自动寻路至长安城驿站
  2. 与驿站老板对话
  3. 选择"传送到大唐国境"
  4. 确认传送
- **识别需求**：
  - 驿站老板 NPC 识别（YOLO）
  - 传送选项识别（OCR）

#### 4.1.4 鬼王任务领取模块
- **功能描述**：自动找黑无常领取鬼王任务
- **处理流程**：
  1. 从大唐国境寻路至地府
  2. 与黑无常对话
  3. 选择领取鬼王任务
  4. 确认领取
- **识别需求**：
  - 黑无常 NPC 识别（YOLO）
  - 对话界面识别

#### 4.1.5 抓鬼任务领取模块
- **功能描述**：自动找钟馗领取抓鬼任务
- **处理流程**：
  1. 从黑无常处寻路至钟馗处
  2. 与钟馗对话
  3. 选择"我要抓鬼"
  4. 确认领取任务
- **识别需求**：
  - 钟馗 NPC 识别（YOLO）
  - 对话界面识别
  - 任务选项识别（OCR）

#### 4.1.6 天眼通符使用模块
- **功能描述**：自动使用天眼通符显示真实坐标
- **处理流程**：
  1. 打开道具栏
  2. 找到天眼通符
  3. 使用天眼通符
  4. 等待坐标更新
- **识别需求**：
  - 道具栏界面识别
  - 天眼通符图标识别
  - 任务栏坐标更新检测

#### 4.1.7 任务识别模块
- **功能描述**：从任务追踪栏（及必要时对话窗口）解析「地图名、坐标、鬼类型」等，供寻路与战斗策略使用。
- **输入**：任务栏 ROI 截图；若启用天眼，则取**使用天眼后**同一 ROI 的截图。
- **处理流程（建议）**：
  1. **ROI 固定**：按分辨率配置裁剪任务追踪区域，减少背景干扰。
  2. **OCR**：PaddleOCR 等；可准备**地名 / 鬼型**白名单字典做后处理纠错。
  3. **正则与规则**：兼容多种文案，例如「前往长寿村(200,150)」「长寿村 200 150」「消灭××鬼」等（**以你提供的真实截图文案为准**扩展规则）。
  4. **多目标**：若同时存在鬼王与普通抓鬼坐标，解析为两个字段或列表，由调度器按当前任务模式选用。
  5. **置信度**：解析失败或坐标非法时重试 OCR、或中止并打日志。
- **输出**：结构化任务数据（见 §5.1）
  ```json
  {
    "ghost_type": "女鬼",
    "location": "长寿村",
    "coordinates": [200, 150],
    "ghost_king_coords": [180, 160],
    "raw_text": "…",
    "confidence": 0.92
  }
  ```
- **非目标（第一版可不做）**：读内存、注入、封包解析。

#### 4.1.8 自动寻路模块
- **功能描述**：自动寻路至任务目标位置
- **实现方式**：
  - ALT+G 坐标寻路（主要）
  - 导标旗传送（可选）
  - 地图切换处理（大唐国境 ↔ 地府）
- **优化策略**：
  - 优先使用导标旗（如果有）
  - 其次使用 ALT+G 寻路
  - 自动处理驿站传送
  - 自动处理大唐国境→地府路线

#### 4.1.9 战斗模块
- **功能描述**：自动进行抓鬼战斗
- **功能**：
  - 检测战斗开始
  - 自动选择攻击技能
  - 自动选择目标（优先主鬼）
  - 检测战斗结束
- **识别需求**：
  - 战斗界面识别（模板匹配）
  - 主鬼识别（YOLO）
  - 回合数识别（OCR）
  - 血量识别（颜色检测）

#### 4.1.10 任务交付模块
- **功能描述**：返回找钟馗交任务
- **处理流程**：
  1. 战斗胜利后返回地府
  2. 自动寻路至钟馗处
  3. 对话交付任务
  4. 领取奖励
  5. 记录任务完成情况
- **识别需求**：
  - 钟馗 NPC 识别（YOLO）
  - 奖励界面识别
  - 奖励物品识别（可选）

### 4.2 辅助功能模块

#### 4.2.1 自动补给模块
- **功能描述**：自动购买补给品
- **功能**：
  - 检测血量/魔法值
  - 检测背包药品
  - 自动购买包子/魔法药
  - 自动使用药品

#### 4.2.2 异常处理模块
- **异常类型**：
  - 战斗死亡
  - 任务超时
  - 掉线
  - 找不到 NPC
  - 组队失败
- **处理策略**：
  - 重试机制
  - 记录日志
  - 用户通知

#### 4.2.3 日志记录模块
- **记录内容**：
  - 任务轮数
  - 战斗回合数
  - 获得奖励
  - 异常情况
  - 完成时间

## 五、数据结构设计

### 5.1 任务信息结构
```python
class GhostTaskInfo:
    task_number: int          # 第几轮（或第几只，与游戏显示对齐）
    ghost_type: str           # 鬼魂类型（若 OCR 可区分）
    location: str             # 地图名（标准化后）
    coordinates: tuple        # 坐标 (x, y)
    ghost_king_coords: tuple  # 鬼王坐标（可选，与玩法开关一致）
    raw_text: str             # OCR 原始文本，便于调试
    confidence: float         # 解析置信度
    need_extra_npc: bool      # 是否需额外 NPC（如版本特殊流程，占位）
    reward_type: str          # 奖励类型（预估，可选）
```

> 说明：旧版字段 `need_luxiao` 与当前文内 NPC 表不一致，已改为泛化字段 `need_extra_npc`；若你版本确有单独交任务 NPC，在 `NPC_CONFIG` 中扩展并在调度器分支即可。

## 十、可实现程度、协作边界与需你方提供的资料

### 10.1 在「屏幕自动化 + YOLO/OCR + 配置驱动」前提下，可交付的能力层级

| 层级 | 内容 | 说明 |
|------|------|------|
| L1 | 工程骨架 | 窗口绑定、截图 ROI、键鼠封装、日志、`NPC_CONFIG` / 路径配置加载 |
| L2 | 跑路 + 领任务 | 固定分辨率下：驿站 → 地府 → 钟馗领任务（可暂不跑鬼王）；任务栏 OCR 出坐标；ALT+G 寻路 |
| L3 | 闭环一轮 | L2 + 战斗内自动操作（攻击 / 法术可配置）+ 回钟馗交付；天眼通符可选接入 |
| L4 | 多轮与策略 | 双倍领取、黑无常鬼王分支、失败重试、超时、简单补给；与现有 `zhuagui` 数据集/YOLO 管线对接 |
| L5 | 多开队长 | 队伍同步、队员挂机假设下的鲁棒性、复杂异常恢复（掉线、死亡、散队） |

**说明**：可协助 **代码、配置结构、识别与状态机设计、训练数据清单与 `dataset/data.yaml` 对齐**；不承诺规避游戏检测，也不代承担封号风险。能达到 L 几 strongly depends on 你是否提供 **一致分辨率下的截图与实测坐标**。

### 10.2 需你方提供（按优先级）

1. **环境与约束**：分辨率（窗口/全屏）、是否五开、操作系统、是否使用外网 OCR。
2. **玩法开关**：是否做鬼王、是否必须用天眼、是否每次领双倍、队长是否全为自己角色。
3. **截图包（强烈建议）**：
   - 任务栏在「领任务后」「使用天眼前」「使用天眼后」各若干张（含不同地图文案）
   - 马副将 / 驿站 / 黑无常 / 钟馗 对话关键步骤截图
   - 战斗界面 1～2 张（用于回合按钮与主怪区域 ROI）
4. **实测坐标**：当前版本下四个（或六个）NPC 的常用站位坐标、驿站选项文案是否变化。
5. **战斗策略**：门派、默认法术或普攻、是否自动加血、药品放置格子或快捷键。
6. **已有资产**：`zhuagui/dataset` 当前类别列表、`data.yaml`、已训练权重路径（若直接集成推理）。

## 十一、风险评估

### 11.1 技术风险
- **NPC 识别率**：战斗场景复杂，识别率可能不稳定
  - *应对*：增加训练数据，迭代优化模型
- **战斗识别**：战斗动画可能干扰识别
  - *应对*：多帧检测，状态确认
- **任务解析**：任务文字格式变化
  - *应对*：正则表达式 + 关键词匹配

### 11.2 使用风险
- **封号风险**：自动化可能被检测
  - *应对*：模拟人工操作，添加随机延迟
- **组队风险**：野队配合问题
  - *应对*：优先自己当队长

### 13.4 `guide_flag` 配置与实现对照表

| 配置项 | 作用 |
|--------|------|
| `park_before_inventory_norm` | 打开背包前鼠标移到窗口内比例，默认中心 |
