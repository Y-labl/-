# MHXY 账本 · 产品说明

> 与 [DESIGN.md](./DESIGN.md)（技术设计）配套；**功能变更时请同步更新本文**。

## 1. 产品定位

帮助《梦幻西游》玩家记录 **物品 / 现金 / 点卡**，并用 **推荐榜** 管理日常与限时活动优先级；支持 **日 / 周 / 月** 总览。

## 2. 每日限时活动从哪来

- 服务端会 **定时拉取** 你在环境变量里配置的 **活动 JSON 地址**（`ACTIVITIES_FEED_URL`），默认约 **每 5 分钟** 更新一次缓存。
- 若未配置地址、网络失败或返回空列表，则自动使用 **内置活动日历**（与当前代码版本一致）。
- 推荐榜页底部会显示当前数据来源（远程 / 内置 / 回退原因）与活动条数，便于确认是否拉取成功。

**远程 JSON 示例**见仓库：`server/data/activities-feed.example.json`。

## 3. 推荐榜与「每日已刷」

| 行为 | 是否写入「每日已刷」 |
|------|----------------------|
| 仅在列表里看到任务，**未点完成** | **否** |
| 在推荐榜点击 **完成** 并确认 | **是** |
| 任务因 **时间结束** 从推荐榜消失，事后在 **补录完成** 提交 | **是** |

- **总览**里「今日已完成任务」、周/月 **完成任务次数** 均只统计上述 **已写入** 的记录。
- 限时活动与模板副本在系统内用不同键区分，同一天不会对同一活动重复记账。

## 4. 补录完成

- 入口：**侧栏 → 补录完成**（`/app/tasks/backfill`）。
- 选择 **业务日** 后，列表展示当日星期所匹配的：
  - 无固定时段的 **模板日常 / 副本**（未记录完成的）；
  - **实时活动**（含已从推荐榜消失的限时活动）。
- 点击 **补录完成** 并确认时间后，与在推荐榜完成一样，记入 **每日已刷**。

## 5. 智能录入（语音 / 截图）

- **语音**：依赖浏览器语音识别（建议 Chrome / Edge），在本地转文字后由服务端 **规则解析**（非大模型）。
- **截图 OCR**：上传到服务器识别，首次可能较慢；可在服务端关闭 OCR（见技术文档）。

## 6. 限时置顶与提醒

- 在活动开始前一段「提前时间」内至活动结束，推荐榜会将对应活动 **置顶**（具体规则见技术文档）。
- 可选开启 **每 10 分钟** 的系统通知，仅在存在置顶活动时提醒（需浏览器通知权限）。
